Frequently Asked Questions

What is the difference between var, let, and const?
In JavaScript, var is function-scoped, hoisted, and allows redeclaration and reassignment. let is block-scoped, hoisted but not initialized, and allows reassignment but not redeclaration. const is also block-scoped, hoisted but not initialized, and prevents reassignment, though object properties can still be modified. It's best to use const by default, let when reassignment is needed, and avoid var.
What is the difference between map(), forEach(), and filter()?
In JavaScript, map(), forEach(), and filter() are array methods used for iteration but serve different purposes. map() returns a new array by applying a function to each element, preserving the original array. forEach() executes a function on each element but does not return a new array, making it useful for side effects like logging or modifying external variables. filter() creates a new array with elements that satisfy a given condition, excluding those that don’t. While map() and filter() return new arrays, forEach() simply iterates without producing a new array.
Explain arrow functions and how they are different from regular functions
Arrow functions provide a concise syntax for writing functions, using the => syntax, and they inherit the this value from their surrounding context, unlike regular functions, which have their own this binding. Additionally, arrow functions cannot be used as constructors, while regular functions can. Arrow functions also omit the need for the function keyword, making them more compact and often used for shorter, inline functions.
Explain how JavaScript Promises work
JavaScript Promises represent the eventual completion (or failure) of an asynchronous operation and its resulting value. A Promise is in one of three states: pending, resolved (fulfilled), or rejected. You can handle the result or error of a Promise using .then() for success and .catch() for errors, providing a way to chain and manage asynchronous code more effectively.
Explain how closures work in JavaScript
Closures in JavaScript occur when a function retains access to its lexical scope, even after the outer function has finished executing. This allows inner functions to access variables from their outer function’s scope, enabling powerful patterns like data encapsulation and function factories. Closures are commonly used for creating private variables and managing state in asynchronous code.
